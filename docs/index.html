Claude Code · operating policy

The strongest model directs. Cheaper models do the work.

A portable policy for Claude Code that puts your best model in the director's seat. It breaks the task apart, routes each piece to the right tier, and checks the result before trusting it. The judgment lives in files, not in any one model.

Hand your best model the clipboard, not the keyboard

One top-tier model doing everything gives the best result per step. It also doesn't scale: no parallelism, its context fills with noise on long jobs, and it bills premium rates to rename a variable. Delegation fixes all three, but only when the delegation is disciplined. This policy is that discipline, written down as files a model reads and follows.

two modes

Full control, or half the cost

Both live in a single chat. You switch tiers with /model; the conversation carries over. No new chat, no copied brief.

/direct default · max quality

The director runs the whole loop

Your strongest model plans, dispatches workers, checks every result, and integrates the pieces. Top-tier judgment on every decision, including the small mid-task ones. This is the default and the highest-quality path.

/plan economy

The director plans, then steps aside

Your strongest model produces the plan and a routing table, then you drop the same chat one tier down to run it. You trade a little mid-task judgment for roughly half the orchestration cost.

Stuck mid-task? Flip back up one tier, ask the hard question, flip down to keep building. Same chat throughout.

the routing table

Three tiers, one rule

A piece with a checklist and a verifiable output routes down. A piece needing judgment, taste, or a decision a human will see stays up.

Frontier

Director only, never a subagent. Plans, verdicts, decisions that must be right, taste-critical client work. Where the intelligence originates.

Mid

Fallback director and heavy worker. Anything a human sees, taste-heavy work, and any piece that came back weak from the floor.

Floor

The workhorse. Coding, edits, research, drafts, searches, audits, bulk work. Nothing runs below it.

Tuned on Fable 5 → Opus 4.8 → Sonnet 5. Drop in your own three tiers.

the director method

Five gates the director runs on itself

The discipline behind the loop. A skill file can't transfer intelligence; it transfers how to scope, gather evidence, attack your own plan, verify, and report. Each gate passes before the next opens.

1

Scope before work

Restate the task and what "done and correct" looks like. Name what's out of scope. Budget effort to the stakes: a lookup gets minutes, an irreversible change gets the full method.

2

Evidence before reasoning

Never design from memory of what a file or API "probably" looks like. Open it, quote the line. Training memory is a hypothesis generator, not a source.

3

Attack your own answer

Before committing, switch sides and try to break it. Write down the likeliest failure mode and its countermove. A plan with no way to fail isn't understood yet.

4

Verify before declaring done

"It ran" is not "it's right." Diff the files, run the test, compare numbers to their source. Never trust a completion report, a worker's or your own. Flag what you couldn't verify.

5

Report with calibration

Lead with the outcome in one sentence. Separate what you verified, what you inferred, and what you assumed. State confidence and what would change your mind.

Install

Copy the files into your Claude Code config at ~/.claude, then open a session.

  • ·Review CLAUDE.md first. Merge it into your existing global instructions rather than overwriting.
  • ·Set your default model. Point it at your frontier tier at high effort so every session opens as a director.
  • ·Swap the tiers. The routing table names the models it was tuned on; replace them with yours.
# from the repo root
cp CLAUDE.md ~/.claude/CLAUDE.md
mkdir -p ~/.claude/commands ~/.claude/skills ~/.claude/reference
cp commands/*.md ~/.claude/commands/
cp -r skills/director-method ~/.claude/skills/
cp reference/dispatch-brief.md ~/.claude/reference/
Judgment in a model is rented. Judgment in a file is owned. Take it, fork it, wire in your own models. That is the whole idea.
